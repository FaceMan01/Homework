SSH протокол (Secure Shell) - это сетевой протокол, который обеспечивает защищенное удаленное управление компьютером и другие сетевые службы через незащищенную сеть, используя криптографические методы шифрования.

Отличие SSH от Telnet:
Безопасность: SSH предлагает шифрование данных, тогда как Telnet передает информацию в открытом, незащищенном виде.
Аутентификация: SSH поддерживает различные методы аутентификации, включая аутентификацию по ключам, в то время как Telnet поддерживает только аутентификацию по паролю.
Использование: SSH является стандартом для безопасного удаленного доступа, в то время как Telnet в современных сетях используется редко и считается устаревшим из-за своей уязвимости.

Способы определения установки пакета OpenSSH:
Использование команды в терминале Linux: dpkg -l | grep openssh (для систем на основе Debian) или rpm -q openssh (для систем на основе RPM).

Способ вывести в консоль текущий статус SSH-сервера:
Для систем на основе systemd: systemctl status sshd

Шаблон настроек для конфигурации сервера SSH:
Файл конфигурации сервера SSH обычно называется sshd_config и располагается в директории /etc/ssh/.

Настройка для ограничения входа по IP-адресу:
Директивы AllowUsers, AllowGroups, DenyUsers, и DenyGroups в sshd_config могут включать указание IP-адресов.

Настройка для запрета входа от имени root:
Директива PermitRootLogin в sshd_config, которую можно установить в значение no.

Настройка для отключения необходимости ввода пароля при подключении:
Установка PasswordAuthentication в no и использование ключей для аутентификации.

Настройка для включения входа по ключу шифрования:
Установка PubkeyAuthentication в yes.

Настройка для ограничения количества удаленных подключений:
Директива MaxSessions в sshd_config.

Работа аутентификации по ключам:
Пользователь генерирует пару ключей (публичный и приватный). Публичный ключ размещается на SSH-сервере в файле .ssh/authorized_keys пользователя.
При подключении к серверу SSH-клиент предлагает использовать приватный ключ для аутентификации.
Сервер проверяет сигнатуру, созданную приватным ключом, используя сохраненный публичный ключ. Если проверка проходит успешно, доступ разрешается без ввода пароля.
